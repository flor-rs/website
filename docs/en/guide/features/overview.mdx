# Framework Capabilities

Framework capabilities section specifically introduces `flor`'s optional capabilities. They are mostly enabled through Cargo features, and after enabling they expose the corresponding API, event builder or platform behavior.

If you just want to look up all feature names, first see [Quick Overview](/en/guide/startup#framework-features). This section explains platform capabilities in detail: what you can do after enabling, where the entry point is, and which tutorials/APIs are related.

```toml
[dependencies]
flor = { version = "0.1.0", features = ["direct2d", "clipboard", "drag-drop"] }
```

## Platform Capabilities

| Feature | Capability | Entry |
| --- | --- | --- |
| `clipboard` | Read/write system clipboard. | [Clipboard](/en/guide/features/clipboard) |
| `drag-drop` | Receive system drag-drop enter, hover, leave, release events. | [Drag-Drop](/en/guide/features/drag-drop) |
| `tray` | Add, update, delete system tray icon, and receive tray mouse events. | [System Tray](/en/guide/features/tray) |
| `theme-change` | Expose system theme change related event entry. | [Theme Change](/en/guide/features/theme-change) |
| `monitor` | Query monitor list, work area, DPI, scaling etc. info. | [Monitor Information](/en/guide/features/monitor) |
| `hi-dpi` | Enable process-level DPI awareness, paired with DPI change event to update layout units. | [High DPI](/en/guide/features/hi-dpi) |
| `cross-thread-window-creation` | Allow requesting window creation outside event loop thread. | [Cross-thread Window Creation](/en/guide/features/cross-thread-window-creation) |

Event-type capabilities usually also need to be paired with [Event Builder](/en/guide/use/builder/event) use; complete handler signature and current dispatch status see [Handler API](/en/api/handler).
