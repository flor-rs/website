# Drag-Drop

`drag-drop` feature is used to enable system drag-drop events. After enabling, widgets can receive drag enter, hover, leave and release through event builder.

```toml
[dependencies]
flor = { version = "0.1.0", features = ["direct2d", "drag-drop"] }
```

## Basic Usage

Drag-drop is event capability, entry in [Event Builder](/en/guide/use/builder/event).

```rust
use flor::base::platform::DropEffect;
use flor::view::builder::EventBuilder;
use flor_lys::label::label;

let drop_area = label("Drag Here")
    .on_drag_enter(|_view_id, _key_state, _pos, formats, effect| {
        if formats.iter().any(|format| matches!(format, flor::base::platform::DragFormat::Files(_))) {
            *effect = DropEffect::Copy;
        }
    })
    .on_drop(|_view_id, _key_state, _pos, data, effect| {
        println!("{data:?}");
        *effect = DropEffect::Copy;
    });
```

Events:

| Method | Purpose |
| --- | --- |
| `on_drag_enter(...)` | Drag content enters widget hit area. |
| `on_drag_over(...)` | Drag content moves on widget. |
| `on_drag_leave(...)` | Drag content leaves widget. |
| `on_drop(...)` | User releases drag content. |

Complete signature and current dispatch status see [Handler API's Drag-Drop Events](/en/api/handler#拖放事件).

## Related Types

Drag-drop events will directly encounter these types:

| Type | Purpose |
| --- | --- |
| `DropEffect` | Widget's drag-drop feedback to system,表示当前是拒绝,复制,移动,链接等效果. |
| `DragFormat` | Data formats available in drag enter and hover阶段,用来判断当前拖进来的内容大概是什么类型. |
| `DragData` | Real data obtained in `on_drop(...)`阶段. |
| `KeyState` | Mouse button and modifier key state when drag-drop occurs. |
| `MousePosition` | Mouse coordinates when drag-drop occurs. |

`on_drag_enter(...)` and `on_drag_over(...)` get `&[DragFormat]`,适合先判断能不能接收; `on_drop(...)` gets `&DragData`,适合读取真正的数据.

## DropEffect

`DropEffect` is used to tell system how current widget准备处理 drag content. Values include:

| Value | Meaning |
| --- | --- |
| `DropEffect::None` | Not accept. |
| `DropEffect::Copy` | Accept, and表示 copy. |
| `DropEffect::Move` | Accept, and表示 move. |
| `DropEffect::Link` | Accept, and表示 create link. |
| `DropEffect::Scroll` |表示 scroll feedback occurred during drag-drop. |

In `on_drag_enter`, `on_drag_over` and `on_drop` can modify feedback through `&mut DropEffect`.

## DragFormat and DragData

`DragFormat` is used for enter and hover阶段,表示 current drag object declares it can provide哪些 formats:

| Value | Meaning |
| --- | --- |
| `DragFormat::Files(_)` | File drag-drop. |
| `DragFormat::Text(_)` | Text drag-drop. |
| `DragFormat::Image(_)` | Image drag-drop. |
| `DragFormat::Custom(_)` | Platform or application custom format. |

`DragData` is used for final release阶段,表示 real data handed to widget:

| Value | Meaning |
| --- | --- |
| `DragData::None` | No available data. |
| `DragData::Files(paths)` | File path list. |
| `DragData::Text(text)` | Text content. |
| `DragData::Image(bytes)` | Image byte data. |
| `DragData::Raw(value)` | Platform raw data, used for extension scenarios. |
